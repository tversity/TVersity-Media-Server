Detailed Description

A media server device. With this class you can browse the content of the media server.

You can get the current servers by calling getCurrentDevices: (TVersityAFAirFlinger)

The server device methods use blocks to return asynchronous result of the invocation. For some details on that, checkout the about blocks paragraph in the SDK main page.

Get the children of one of the child objects of this server.

The child objects can be containers or items.

The block is called asynchronously in the main thread of the application.

Parameters
[in]blockThe block to invoke with the children information. The block is called asynchronously in the main thread of the application.
[in]objectThe container object of this server for which we want to get the children. If nil is passed, returns the children of the root of the server.
[in]startIndex0 based index from where to start the children fetch
[in]fetchCountMaximum number of children to fetch. Use 0 to fetch all children.
Notice there is a limit on the number of fetched objects which varies depending on the amount of metadata returned. Generally a value of up 100 should return without an error.
[out]outErrorOn return, if an error occured, outError will contain an error object with error information.

This method can return an NSError object with an error code in outError parameter, or asynchronously in the error parameter of the block.
In either case, the error code will be one of the following:

  • TVersityAFInvalidDeviceError No device is known with the specified id
  • TVersityAFDeviceOfflineError The device is offline and cannot be accessed
  • TVersityAFInvalidCommandError The device does not support this command or its current state does not allow it
  • TVersityAFOutOfBoundsError The response is too long. Try again with a lower fetchCount. This error will be returned only in the block.
  • TVersityAFUnknownError
Returns
YES if the method succeeds. NO if an error occured.
