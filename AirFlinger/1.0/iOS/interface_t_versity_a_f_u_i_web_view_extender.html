Detailed Description

Attaching TVersityAFUIWebViewExtender object to a UIWebView instance, provides additional functionality related to AirFlinger.

This object sends to the mediaDelegate events from an html5 video element (if one exists on the current web page) through the TVersityAFUIWebViewMediaDelegate protocol.

Remarks
If more than one video element is found on the page, events will be sent from the last video element that started playing.
For example, assuming there are 2 videos A and B on a page.
When you start playing video A, you will get a webViewPlay event from video A element with the url being played.
If you now start playing video B, you will get the webViewPlay event from video B element.
Switching back to video A and clicking resume (or pause and resume if it was not paused), will result in getting webViewPlay event again from video A, since the last video started playing is now video A.

Initializes this object by passing a UIWebview to connect to and a media delegate to call when video element events happen.

Parameters
[in]webViewThe UIWebView object to attach to.
[in]delegatean object that implements the TVersityAFUIWebViewMediaDelegate protocol that will receive extended AirFlinger events from the UIWebView object.

NOTICE: You should initialize this object after assigning a delegate to the UIWebView object (assuming you need to assign one). Otherwise you will not get events from this object. This is because this object also passes itself as a delegate to the UIWebView. It first handles itself calls to the delegate methods and then passes the calls to the previous object that set itself as delegate of the UIWebView.
