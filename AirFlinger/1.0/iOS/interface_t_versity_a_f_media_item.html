Detailed Description

An abstract base class for a media item.

This class represents a media item on a local iOS device or a remote media server. A media item represents an object (e.g. an image or an audio file) that can be displayed or played on a DMR (Digital Media Renderer). A media item may have more than one resource attached to it.

Method Documentation

- (BOOL) artWithBlock: (TVersityAFImageDownloadCompletedBlock)  block

Fetch the art of the object asynchronously.

If the art property returns nil, use this method to try getting the art asynchronously. For a remote media item for example, this method will download the art from the server. When the art is downloaded, the block is called in the main thread of the application.

Parameters
blockThe block to be called when the art is downloaded. In case there is no art, the block will be called with a nil image and a nil error. The block will be called on the main thread of the application.
Returns
Returns YES if the art is being fetched, or if there is no art. NO in case of an error, or if the block parameter is nil. In case of an error, the error parameter in the block contains the error information.

Initialize a media object.

Usually you will get media objects from the SDK, so you will not need to use this method.
Use this method in case you obtain on your own links to resources on a remote media server and want to use them with the SDK.
In case you obtain on you own assets from iOS, a better way to initialize a media item is using TVersityAFLocalMediaItem#mediaItemWithAsset:

Parameters
nameThe media object name
typeMedia object type
objectIdAn id of the media object
resourcesAn array of TVersityAFResource objects. You should initialize these objects as well.
iconThe icon representing the object
artThe full screen image representing the object
Returns
The initialized object, or nil if initialization failed.
