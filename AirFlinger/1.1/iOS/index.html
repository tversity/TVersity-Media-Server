TVersity AirFlinger iOS SDK Documentation

The TVersity AirFlinger SDK provides all the necessary building blocks to add media shifting capabilities to mobile apps and more specifically to create engaging multi-screen experiences. It's main features are media shifting of local/remote/web content, screen mirroring, and control plus synchronization during playback.

Using this SDK you can:

  • Discover devices on the network and interact with those devices.
  • Stream content from various sources including the web, the host device (device on which the SDK is running) and other sources.
  • Control playback of streamed content including play, pause, stop, seek, volume controls and so on.

We distinguish between two types of devices, players and servers:

  • Players can receive media over the network and render it.
  • Servers represent a source of content with the capability to provide metadata with respect to the content they host as well as deliver that content over the network.

The SDK encapsulates multiple industry standards, thus allowing a wider range of devices to be supported.
Each standard utilizes a different terminology for a players and a server.
For example, the UPnP and DLNA standards refers to them as Media Renderer and Media Server while the Miracast standard refers to them as Sink and Source devices.

This Objective-C SDK is designed for developing applications for the iOS operating system.
It is thread safe, but is designed for being used from a GUI application without the need to handle threads or network related issues.
Therefore, whenever a method of this SDK should return an asynchronous response, it will be sent in the main thread of the application.

To start using the SDK, begin with the TVersityAFAirFlinger class.
Using this class you initialize the SDK and get a collection of media devices identified on the network.
Once you have the collection of devices, you can control them or query them using the subclasses of the TVersityAFDevice.
With a TVersityAFPlayer class, you can get information and control playback of media on a media renderer device.
With a TVersityAFServer class, you can get information and browse the media available on a media server.

Other capabilites provided by this SDK are in helper classes which extend iOS to provide easier access to media items, or add features related to "media Flinging" to existing components.
Check the Helpers modules for more details.

About blocks: A common practice is the SDK is to use blocks for returning a result in case of asynchrouns invocations.
A member function that accepts a block as a paremeter, returns immediately.
The block will be called asynchronously with the result of the asynchronous invocation of the method.
Invocations of result blocks always occur in the main thread of the application.
Notice that such methods can fail either in the synchronous part or the asynchronous part of the invocation.
For example, if you call playURI method of TVersityAFPlayer, this method can fail in the synchronous part if for example you pass an invalid URI parameter. In that case it will return the error immedialty.
If however everything is valid, the synchronous part will return without an error. The command will then be sent to the player device.
The player will return an asynchronous response to this command which can contain an error, if for example the URI could not be accessed.

About memory management: The SDK supports only ARC memory management. Using manual memory management with the SDK is not recommended.
