Detailed Description

The protocol that a delegate of TVersityAFUIWebViewExtender should implement in order to get events from an HTML5 video element in a page loaded by a UIWebView.

Video playing ended event.

This event is called when video playing had reached the end of the video. The source of this call is an onended event of the video element as defined by html5 specification.

A new Video started or resumed playing event.

This event is called when the video element changes state from stopped to playing, or in the case of more than one video element on a page, when the last video that started playing had changed.
The source of this call can be an onplay event of the video element as defined by html5 specification, or the TVersityAFUIWebViewExtender.

Parameters
webViewThe UIWebView containing the page with the video element.
urlThe url that is being played (the value of src attribute of the video element).
- (void) webViewPlaying: (UIWebView *)  webView
currentTime: (float)  time 
optional

Video resumed playing event.

This event is called when video playing is resumed. It is called when play resumed after a pause, but also after play started from stopped state. The source of this call is an onplaying event of the video element as defined by html5 specification.

Remarks
This event may in some cases be called immediatly after a webViewPlay:url: event.
Parameters
webViewThe UIWebView containing the page with the video element.
timeThe current time position in the video, in seconds.
