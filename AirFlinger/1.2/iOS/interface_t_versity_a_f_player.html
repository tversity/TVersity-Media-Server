Detailed Description

The Player device provides the functionality for playing media, getting information about the media and getting information about the player device.

You can get the current players by calling getCurrentDevices: (TVersityAFAirFlinger)

Many of the player device methods use blocks to return asynchronous result of the invocation. For some details on that, checkout the about blocks paragraph in the SDK main page.

Sends a URI to play.

This method returns the result of the command asynchronously.

Parameters
[in]uriThe uri to play. You can pass a URL of a media from the web, or use the helper method getURLFromAsset: to obtain the URI from an asset of the OS.
[in]titleThe title of the media to play. Can be nil.
[in]mimeTypeMime type of the media to play. Can be nil.
[in]blockA block to be called asynchronously with the result of the command. The block is called in the main thread of the application.
[in]posBlockA block to be called asynchronously during playback with position information. Can be nil. The block is called in the main thread of the application.
[out]outErrorOn return, if an error occured, outError will contain an error object with error information.

This method can return one of the following error codes:

  • TVersityAFInvalidDeviceError No device is known with the specified id
  • TVersityAFDeviceOfflineError The device is offline and cannot be accessed
  • TVersityAFInvalidCommandError The device does not support this command or its current state does not allow it
  • TVersityAFInvalidParamError The uri parameter is not valid. Only HTTP or AirFlinger URLs are acceptable. AirFlinger ULRs can be extracted from an iOS asset using the getURLFromAsset: helper method.
  • TVersityAFUnknownError
Remarks
The mimeType parameter is optional. If nil is used, the player will try to derive the mime type from the URI's extension.
Returns
YES if the method succeeds. NO if an error occured.
