Detailed Description

This is the main AirFlinger class. With this class you start, initialize and terminate the AirFlinger SDK. Through it you will get access to player and server devices identified on the network.

You must call initInstance method before using any other method of this class. You must call terminate when you are done with using the class.

This class should never be instantiated directly. To get an instance use the getInstance method.
Before starting to use the instance you should initialize it with a call to initInstance.
Therefore initInstance will usually follow the first call to getInstance.

Method Documentation

- (NSArray *) getCurrentDevices: (TVersityAFDeviceKinds)  deviceKind

Get available devices.

Parameters
[in]deviceKindDefine the kind of devices you want to get.
Returns
An array of TVersityAFDevice objects, or nil if no device was found. Each object can be cast to TVersityAFPlayer or TVersityAFServer after type checking. If deviceKind was passed, only devices of the requested kind will be returned, namely, TVersityAFPlayer or TVersityAFServer objects.

Important: Once a device is added to the list, it is never removed, even if it turned offline.
A device can turn offline if it was disconnected or shutdown, and it can turn online again later.
Use the delegate passed to the initInstance method for real time monitoring of changes in the available devices.

Parameters
[in]listenerPortThe port on which the class should listen for communication with devices.
[in]delegateA delegate object for getting update messages from this class. For example if you pass TVersityAFEventPlayerStatusChange in the eventMask, the delegate will be notified on changes in the status of player devices on the network.
[in]eventMaskA mask of TVersityAFEventTypes values defining the events you want to monitor. Notice that the value TVersityAFEventPositionInfo is ignored if used with this mask. This event kind can be monitored using the TVersityAFPlayer class.
Returns

- (BOOL) searchDevices

Call this method to instruct the SDK to refresh the list of devices identified on the network.

You do not have to call this method in order for the device list to refresh. The SDK keeps updating the device list in the background. However, you may choose to initiate this refresh at a specific time when you want to make sure the device list is the most updated for that time.

Returns
YES or NO based on success / failure.
- (TVersityAFResource*) selectResourceForPlay: (TVersityAFMediaItem *)  item
onPlayer: (TVersityAFPlayer *)  player 

Use this method to check with the sdk which resource it would select for playing.

A media item may have more than one resource. This method returns the resource that the sdk will select for playing on a player. Use this method when you need to access this resource information. When a player is provided, the selected resource will be the one most fitting to the player's capabilities. If a player is not provided, the best resource to represent this media item will be selected.

Parameters
[in]itemThe media item
[in]playerThe player on which the media item is intended to be played. Optional.
Returns
The selected resource, or nil.
